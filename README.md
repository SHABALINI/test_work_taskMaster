# test_work_taskMaster
Тестовое задание по созданию консольного приложения "todo-лист", выполненное Шабалином М.А.

С чего начать работу?
Скачайте файл "task_master.cpp". Создайте к нему пустой текстовый файл с названием "tasks"(программа принимает только "tasks").
Откройте терминал или командную строку в папке с файлом.

Компиляция и запуск на Linux:
g++ -std=c++11 -o myTaskMaster task_master.cpp
./myTaskMaster

Компиляция и запуск на Windows:
g++ -std=c++11 -o myTaskMaster.exe task_master.cpp
myTaskMaster.exe

Какая структура используется в текстовом файле?
Структура файла:
TASK|{номер задачи}|{Название задачи}|{дата дедлайна}|{Статус (todo - невыполнена, done - выполнена}|{приоритет от 1 до 10}|{Комментарий пользователя}
SUBTASK|{номер задачи.номер подзадачи}|{название подзадачи}|{статус}
Пример:
TASK|1|Купить продукты|2025-06-10|todo|3|Не забыть пакеты
SUBTASK|1.1|Молоко|todo
SUBTASK|1.2|Хлеб|done
TASK|2|Сдать отчет|2025-06-15|todo|1|Проверить формат

Примечание: из-за особенности структуры доступны только однострочные комментарии и запрещено использование символа |, иначе это приведет к сбою в структуре файла.

Программа работает с двумя файлами: "tasks" и "archive". В архиве хранятся удаленные выполненные задачи.

Особенности работы:
*Нумерация задач
Номера задач всегда идут по порядку: 1, 2, 3...

При удалении задачи все последующие задачи автоматически перенумеровываются

При удалении подзадачи они также перенумеровываются внутри задачи

(Подзадачи
Если задача отмечена как выполненная, все её подзадачи также становятся выполненными

Если добавить подзадачу в уже выполненную задачу, она создаётся выполненной

При возврате задачи в статус "не выполнена" подзадачи не меняются

*Дедлайны
Формат даты: YYYY-MM-DD (например, 2025-12-31)

Некорректные даты нормализуются (месяц > 12 → 12, день корректируется под месяц)

При выводе показывается количество оставшихся дней

Задачи без дедлайна отображаются как "Нет дедлайна"

*Архив
Выполненные задачи перемещаются в архив командой clear done

Архив можно просмотреть командой archive

Поиск в архиве работает аналогично поиску по задачам

Задачи в архиве имеют свои исходные номера (не перенумеруются)

*Комментарии
Ограничение: не более 150 символов

Отображаются при выводе списка задач

Если комментарий не задан, строка с комментарием не выводится


Функционал программы:
*open (название файла) - открытие файла для чтения. В файле должна быть нужная структура данных.

*add task <название> <дедлайн (до какого числа)> <приоритет (от 1 до 10, где 1 - самый высокий, а 10 - самый низкий> - добавить задачу. У каждой задачи свой порядковый номер
*add subtask <номер задачи> <название> - добавить подзадачу. У каждой подзадачи номер - это номер задачи+номер подзадачи. Например у задачи №1 есть подзадача №1.1 и №1.2.

*list num - вывести все задачи с номерами дедлайнами и приоритетами, с подзадачами, статусом и комментарием в порядке возрастания по номерам задач.
*list priority - вывести все задачи с номерами дедлайнами и приоритетами, с подзадачами, статусом и комментарием в порядке возрастания по приоритету (сначала самые приоритетные)
*list deadline - вывести все задачи с номерами дедлайнами и приоритетами, с подзадачами, статусом и комментарием в порядке возрастания по дедлайну (сначала те, до которых осталось меньше всего времени)
*list status - вывести все задачи с номерами дедлайнами и приоритетами, с подзадачами, статусом и комментарием в порядке возрастания по статусу (сначала выполненные)
*list statusN - вывести все задачи с номерами дедлайнами и приоритетами, с подзадачами, статусом и комментарием в порядке убывания (сначала невыполненные)

*done task <номер> - отметить задачу выполненной. Все подзадачи становятся автоматически выполненными
*done subtask <номер задачи> <номер подзадачи> - отметить подзадачу выполненной.

*undone task <номер> - возвращает задаче статус "не выполнена". С подзадачами ничего не делается
*undone subtask <номер задачи> <номер подзадачи> - возвращает подзадаче статус невыполненна

*delete task <номер> - удалить задачу
*delete subtask <номер задачи> <номер подзадачи> - удаляет подзадачу

*comment task <номер> - добавить описание/комментарий задачи (до 150 символов). Этот комментарий будет справа от задачи при команде list.

*search task (название задачи) - команда, чтобы искать задачи в списке задач. Выдает все совпадения с названиями игнорируя регистр. (Например если было введено "сол", то в качестве совпадения будет выведено "Загореть на СОЛНЦЕ")

*all done - сделать все текущие задачи выполненными
*clear done - очищает список выполненных задач с подтверждением. Очищенные задачи отправляются в архив.
*clear all - очищает ваще весь список задач с подтверждением. Удаленные задачи просто удаляются без отправления в архив.

*archive - открывается архив всех удаленных выполненных задач
*search arch (название задачи) - команда только для архива, чтобы искать задачи в архиве. Выдает все совпадения с названиями игнорируя регистр. (Например если было введено "сол", то в качестве совпадения будет выведено "Загореть на СОЛНЦЕ")

*cmd - вывод всех возможных команд для взаимодействия с программой.
